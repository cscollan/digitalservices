Alpha stage comes after discovery, lasts around 8 weeks and during this phase we will:

  • build prototypes of the service
  • test prototypes with real users
  • test hypothesis and assumptions
  • determine if the service can be technically built

Start Alpha with an inception workshop. This should:

  • describe what the service is
  • describe what the service does
  • define who the users of the service are
  • confirm what user needs are being met
  • set clear goals for alpha
  • identify risks to be mitigated
  • set a vision for what done will look like
  • outline how legacy services/process will be decommissioned or integrated
  • a backlog for iteration 1
  • set up show and tell and a retrospective for the end of iteration 1

By the end of alpha, we should:

  • know whether to move into beta
  • know what needs to be built in the beta phase
  • have a set of user stories based on journey, not features
  • have a plan for the beta phase and the start of a plan for live
  • have defined how to measure the success of the service through metrics
  • have a plan to decommission legacy services or processes
  • have a completed accessibility audit
  • have a plan for assisted digital

An alpha assessment should also be completed before moving to beta.

Phases

  • Discovery
  • Alpha
  • Beta
  • Live
